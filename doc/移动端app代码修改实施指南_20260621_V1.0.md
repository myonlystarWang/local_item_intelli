## 修改项 1：实现摄像头扫码（核心功能）

### 目标

### 涉及文件

#### [NEW] `lib/screens/scan_screen.dart`
新建一个独立的扫码页面，使用 `MobileScanner` Widget：
- 全屏摄像头预览 + 扫码框 overlay
- 扫码成功后回调 `BarcodeCapture`，提取编码字符串
- 成功识别后自动关闭扫码页并返回编码值给调用方
- 底部提供"手动输入编码"按钮（容错回退，见修改项 2）

核心代码结构：
```dart
import 'package:mobile_scanner/mobile_scanner.dart';

class ScanScreen extends StatefulWidget { ... }

class _ScanScreenState extends State<ScanScreen> {
  final MobileScannerController controller = MobileScannerController();
  bool hasScanned = false; // 防止重复触发

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('扫码识别')),
      body: Stack(
        children: [
          MobileScanner(
            controller: controller,
            onDetect: (BarcodeCapture capture) {
              if (hasScanned) return;
              final barcode = capture.barcodes.firstOrNull?.rawValue;
              if (barcode != null) {
                hasScanned = true;
                Navigator.pop(context, barcode); // 返回识别到的编码
              }
            },
          ),
          // 扫码框 overlay UI
          _buildScanOverlay(),
        ],
      ),
      // 底部：手动输入按钮
      bottomNavigationBar: SafeArea(
        child: TextButton(
          onPressed: () => _showManualInput(context),
          child: const Text('识别失败？手动输入编码'),
        ),
      ),
    );
  }
}
```

  showDialog(
    context: context,
    builder: (ctx) => AlertDialog(

          },
          child: const Text('确认'),

## 修改项 5：身份确认页面（登录替代方案）

### 目标
实现轻量级身份确认机制：作业人员在首次启动时选择姓名和所属大队，所有操作日志自动关联当前身份，以支持后台按人员/大队维度进行统计分析和报表导出。

> [!IMPORTANT]
> 这不是传统的账号密码登录，而是一个“身份声明”页面。因为 App 在无网环境下工作，无法连接服务端验证凭据。目标是**操作可追溯**，而非访问控制。

### 流程设计
1. `main.dart` 启动时检查本地 SQLite 是否已有保存的身份信息
2. **无身份** → 跳转至 `IdentityScreen`（强制，不可跳过）
3. **有身份** → 直接进入 `HomeScreen`，顶部显示当前操作人标识

### 涉及文件

#### [NEW] `lib/screens/identity_screen.dart`
身份确认页面，包含：
- 标题：“操作人身份确认”
- 两个 `DropdownButtonFormField`：“操作人姓名”和“所属大队”（从本地字典读取）
- 确认按钮：写入本地 SQLite 的 `settings` 表，然后跳转到 `HomeScreen`

```dart
class IdentityScreen extends StatefulWidget { ... }

class _IdentityScreenState extends State<IdentityScreen> {
  String? selectedName;
  String? selectedTeam;
  List<String> operators = [];
  List<String> teams = [];

  @override
  void initState() {
    super.initState();
    _loadDicts();
  }

  Future<void> _loadDicts() async {
    final ops = await LocalDatabase.instance.getDictionaryValues('operator');
    final tms = await LocalDatabase.instance.getDictionaryValues('team');
    setState(() {
      // 字典为空时提供默认列表（暂未同步时的回退）
      operators = ops.isNotEmpty ? ops : ['张建国', '李志刚', '王超', '赵强'];
      teams = tms.isNotEmpty ? tms : ['川庆钻探一队', '中原石油三队'];
    });
  }

  Future<void> _confirm() async {
    if (selectedName == null || selectedTeam == null) return;
    await LocalDatabase.instance.saveSetting('operator_name', selectedName!);
    await LocalDatabase.instance.saveSetting('operator_team', selectedTeam!);
    if (mounted) {
      Navigator.pushReplacement(
        context,
        MaterialPageRoute(builder: (_) => const HomeScreen()),
      );
    }
  }
}
```

新增两个方法：
```dart
Future<void> saveSetting(String key, String value) async {
  final db = await instance.database;
  await db.insert('settings', {'key': key, 'value': value},
    conflictAlgorithm: ConflictAlgorithm.replace);
}

Future<String?> getSetting(String key) async {
  final db = await instance.database;
  final maps = await db.query('settings', where: 'key = ?', whereArgs: [key]);
  if (maps.isNotEmpty) return maps.first['value'] as String;
  return null;
}
```
