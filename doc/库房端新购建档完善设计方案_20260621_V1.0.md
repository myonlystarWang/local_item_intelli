# 库房端新购建档完善设计方案

本方案旨在解决库管员在库房管理端手动建档时品类受限、无法录入耗材零配件的问题。

## 1. API 服务端 (Server) 修改设计

- **数据校验层 (`schemas.py`)**：
  保持原有的 `AccessoryCreate` 和 `AccessoryResponse` 不变，确保可以直接接收新建配件请求。
- **路由接口层 (`main.py`)**：
  新增 `POST /accessories` 接口，用于手动录入新增零配件的初始台账并执行排重校验：
  ```python
  @app.post("/accessories", response_model=schemas.AccessoryResponse)
  def create_accessory(acc: schemas.AccessoryCreate, db: Session = Depends(get_db)):
      db_exist = db.query(models.Accessory).filter_by(barcode=acc.barcode).first()
      if db_exist:
          raise HTTPException(status_code=400, detail="该配件唯一条码在数据库中已存在")
      created = models.Accessory(
          barcode=acc.barcode, name=acc.name, spec=acc.spec,
          unit=acc.unit, safety_stock=acc.safety_stock, current_stock=acc.current_stock
      )
      db.add(created)
      db.commit()
      db.refresh(created)
      return created
  ```

## 2. 库房管理端 (Web) 修改设计

- **API 接口封装 (`api.ts`)**：
  在前端封装中加入 `createAccessory` 方法，在请求失败（演示/Mock 模式下）自动降级写入本地 Mock 列表：
  ```typescript
  async createAccessory(acc: Accessory) { ... }
  ```
- **手动建档界面 (`Lifecycle.vue`)**：
  1. **精密工具建档优化**：
     将原来写死的 `<select>` 选择框改造为 `<input type="text" list="tool-presets">`。既保留了“电动坐封工具”和“阿瓦隆桥塞”的快捷双击提示，又支持库管员手动键入任何全新工具名称（例如“机械坐封活塞”），完全消除硬编码限制。
  2. **新增零配件建档入口**：
     - 在台账页顶部动作按钮区新增“➕ 零配件手动建档”按钮。
     - 点击后弹出 `showCreateAccessory` 弹窗。
     - 字段包含：唯一识别条码（二维码/条形码）、配件名称、规格型号描述、单位（个/套/片等）、安全警戒水位、初始当前库存。
     - 确认后调用 `createAccessory` 保存。
