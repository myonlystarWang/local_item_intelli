## 1. 现状问题总结与技术原因分析

### 1.1 条码与数字码扫描兼容问题
* **问题描述**：条码能正常扫描，但纯数字码/一维码扫描识别失败。
* **原因分析**：App 端使用的扫码插件（如 `mobile_scanner`）在初始化时，可能默认只开启了 QR Code（二维码）或部分特定一维码格式。
* **解决方案**：在扫码配置中，显式指定 `formats` 为支持所有一维码与二维码格式（如 `BarcodeFormat.all`），以增强兼容性。

### 1.2 App 首页 UI 布局问题
* **a. 设置页面大队选项不全且无法滚动**：
  * **原因**：`DropdownButtonFormField` 的下拉弹窗高度受限，或是从数据库读取到的字典数据在过滤去重后，默认的下拉高度没有自适应，导致滚动受阻。
  * **解决方案**：优化下拉菜单配置，调整下拉渲染逻辑并确认字典的完整性。
* **b. 同步后「操作日志」高度过小**：
  * **原因**：布局中对日志区域（ListView）使用了硬编码固定高度，在同步完新数据后，日志数量增多，但容器没有弹性拉伸。
  * **解决方案**：采用 `Expanded` 或 `Flexible` 弹性权重布局，自适应剩余高度。
* **c. 同步后「资产预览」高度过小且无法点击**：
  * **原因**：预览区域高度受限。未对卡片组件（Card）绑定点击路由跳转。
  * **解决方案**：允许资产列表在整体页面中顺畅滑动，并为资产卡片绑定点击手势 `GestureDetector`，跳转至 `DetailScreen`（工具履历详情页）。
* **d. 手动输入工具编码缺少模糊匹配点选**：
  * **原因**：当前的输入框为普通 `TextFormField`，没有与本地数据库的工具编码列表进行联动。
  * **解决方案**：引入 Flutter 原生的 `Autocomplete<String>` 组件，根据用户输入的字符实时在本地过滤出匹配的条码列表供点选。

### 1.3 Web 端台账模块缺失
* **a. 缺少配件台账主表**：
  * **原因**：目前 Web 端重点放在了“精密工具”的生命周期展示，缺乏对易耗配件的独立管理维度。
  * **解决方案**：在 Web 端左侧导航中新增「配件库存台账」页面，对接后端的 `/accessories` 接口，提供独立的库存、规格及补货管理面板。
* **b. 缺少基础字典台账（井号、大队、人员）**：
  * **原因**：基础数据在后端数据库虽然存在，但 Web 端没有暴露管理入口，管理员无法维护。
  * **解决方案**：在 Web 端新增「基础数据维护」模块，提供井号、作业大队、操作人员的新增、编辑与删除操作。

### 1.4 工具操作履历展示顺序问题
* **问题描述**：点击工具查看生命周期，履历记录为正序（最旧的在最上面）。
* **原因分析**：后端返回或前端渲染时，按照时间戳升序（ASC）排列了履历日志。
* **解决方案**：在数据库查询或前端数组渲染时，将履历日志按时间戳降序（DESC）排序，确保最新动态最先展现。

### 1.5 局域网同步校验日志滞后（假数据问题）
* **问题描述**：Web 端「局域网同步校验日志」模块显示的仍是写死的假数据，没有反映最近的同步结果。
* **原因分析**：Web 端使用的是 Mock 列表，未对接后端的真实同步日志 API。
* **解决方案**：在后端 `models` 中引入 `SyncLog` 表记录真实同步事件，并在 Web 端增加轮询或实时获取接口，渲染真实数据。

### 1.6 UI 数据自适应问题
* **问题描述**：数据多或少时，部分区域显示异常。
* **原因分析**：过多使用了硬编码的绝对高度和宽度，没有使用 `LayoutBuilder` 和 Flex 的自适应权重。
* **解决方案**：全面应用「空状态占位图」（Empty State），并在列表外层统一包裹滑动容器（`SingleChildScrollView` 或 `CustomScrollView`）杜绝溢出。

## 2. 优化实施计划

### 第一阶段：App 端体验及漏洞修补（Flutter）
1. 升级扫码组件支持所有条码/二维码格式。
2. 修复手持设置页面下拉项的完整性与滚动体验。
3. 将首页的资产预览和操作日志改造为自适应弹性布局，并为资产预览项绑定点击详情事件。
4. 引入输入框模糊搜索提示（`Autocomplete`）。

### 第二阶段：数据流与展示对齐（Python 后端 + Flutter + Web）
1. 后端查询工具履历改为按时间戳倒序输出。
2. 后端建立真正的同步历史表，供 Web 端拉取真实的「局域网同步日志」。
3. Web 前端对接此日志接口。

### 第三阶段：后台管理页扩展（Vue Web 端）
1. 开发「配件台账主表」管理页面。
2. 开发「基础数据管理」（井号、大队、人员）页面。
