# 移动端功能完善设计方案

本项目移动手持端基于 Flutter 开发，需在离线状态机架构下完成功能完善。本方案旨在解决摄像头真实扫码、手动输入容错、离线操作人身份追溯、预计归还天数可调、本地字典数据写入 Bug 修复及无用依赖清理等问题。

## 1. 数据库升级设计 (SQLite)
为了支持终端本地参数的离线存储（如当前操作人、所属大队、同步服务器地址、终端 UUID 等），需在 SQLite 数据库中引入 `settings` 配置表。

- **表结构设计**：
  ```sql
  CREATE TABLE settings (
    key TEXT PRIMARY KEY,
    value TEXT NOT NULL
  )
  ```
- **版本升级 (DB Version 2)**：
  将 `LocalDatabase` 数据库版本从 `1` 提升至 `2`。
  实现 `onUpgrade` 逻辑，在旧版本升级时，如果 `settings` 表不存在则予以创建：
  ```dart
  Future _onUpgrade(Database db, int oldVersion, int newVersion) async {
    if (oldVersion < 2) {
      await db.execute('''
        CREATE TABLE settings (
          key TEXT PRIMARY KEY,
          value TEXT NOT NULL
        )
      ''');
    }
  }
  ```
- **新增接口封装**：
  - `saveSetting(String key, String value)`
  - `getSetting(String key)`

## 2. 身份确认页面与强制确认机制
为了实现井口操作责任的可追溯性，App 引入“身份声明”模式（在无网环境下通过声明身份绑定操作日志）。

- **强制确认流程**：
  在 `main.dart` 启动时，使用 `FutureBuilder` 检查本地 SQLite 是否已写入 `operator_name` 与 `operator_team`。
  - 若已设置，直接进入 `HomeScreen`。
  - 若未设置，强制跳转至 `IdentityScreen`（不可跳过），选择身份写入本地数据库后再进入 `HomeScreen`。
- **页面设计 (`IdentityScreen`)**：
  - 读取字典表中的 `operator` 和 `team` 数据填充下拉选择框。
  - 若字典尚为空（比如新设备初次启动未同步），提供默认候选数据（如“张建国、川庆钻探一队”等）作为回退方案。
- **操作日志绑定**：
  - 移除 `DetailScreen` 中出库领用表单的原“责任领用人”和“作业队组”下拉框，改为显示“当前操作人：张三（所属大队）”。
  - 出库、地点变更、维保保养等所有本地写入的离线日志中，其 `operator` 字段均自动填充当前设置的姓名。
  - 日志中 `detail` JSON 里新增大队名称字段：`team`。

## 3. 设置页面与配置管理
- **页面设计 (`SettingsScreen`)**：
  在主页 AppBar 增加“设置”入口，进入设置页面后支持以下操作：
  - **身份切换**：支持重新下拉选择姓名和大队，覆盖保存到 `settings` 中。
  - **同步配置**：读取与修改“局域网同步服务器地址”和“手持终端 UUID”。
- **同步页面配置化 (`SyncScreen`)**：
  修改局域网近场同步逻辑，不再使用硬编码的 IP 地址和终端 UUID，而是通过 `LocalDatabase.instance.getSetting` 动态读取，若为空则回退到原默认值。

## 4. 摄像头扫码与手动输入容错
- **摄像头扫码页面 (`ScanScreen`)**：
  使用 `mobile_scanner` 库的 `MobileScanner` 组件实现真实扫码。
  - 自定义扫码界面：加设全屏预览、遮罩框（Scan Overlay）、防止重复触发的控制变量。
  - 扫码成功后，通过 `Navigator.pop(context, barcode)` 返回物理条码。
- **手动输入容错**：
  在扫码页面底部提供“手动输入编码”选项。点击后弹出对话框输入资产编号，确认后同样关闭扫码页并返回输入的资产编号。
- **主页与维保页联动**：
  - 主页点击“扫码”直接跳转至 `ScanScreen` 并等待返回结果，接收到结果后跳转到详情页。
  - 维保页面配件消耗区域增加“扫码添加”图标按钮，点击后跳转至 `ScanScreen` 扫描配件三防标签，成功后在列表中检索自动匹配选中。

## 5. 归还天数 Slider 控件
在 `DetailScreen` 出库领用登记表单中，将硬编码的 `returnDays = 30` 替换为可视化 `Slider` 控件：
- 范围：7 天 - 90 天
- 步长：1 天
- 可实时改变 `returnDays` 状态，并将其正确写入出库离线日志中。

## 6. 代码质量修复与依赖清理
- **修复 forEach 数据库写入 Bug**：
  在 `LocalDatabase.performSyncAlignment` 中，将 `dicts.forEach` 替换为 `for (final entry in dicts.entries)` 的 `for...in` 循环，确保 `await txn.insert` 拥有正确的异步等待，彻底防止同步后字典数据丢失。
- **清理无用依赖**：
  在 `pubspec.yaml` 中彻底移除未使用的 `fl_chart` 依赖，以减小包体积。
