## 1. 现状问题与设计重构说明

### 1.1 网络状态动态监听（解决顶部图标不刷新问题）
* **现状**：手机网络断开或与局域网后端服务断开时，顶部的连接状态按钮无法随之更新，给操作员带来“系统在线”的误导。
* **技术重构**：
  * 使用 `connectivity_plus` 或 `http` 进行轻量级心跳检测（Ping 后端 `/tools` 接口），心跳频率为每 5 秒一次。
  * 根据网络状态，动态更新状态值。当能连通后端时，顶部图标呈现绿色 Wi-Fi 样式且显示 “局域网在线”；若连不通，呈现灰色 Wi-Fi 样式且显示 “离线状态”。
  * 目前顶部绿色 Wi-Fi图标是可点击的，但我不理解为什么要点击，应该根据网络状态变化并不可点击

### 1.2 首页大滚动视图重构（解决组件高度被限死问题）
* **现状**：由于对底部的「资产预览」和「同步校验日志」列表设置了硬编码的固定容器高度，导致在一屏显示下能展示的内容极少，且容易发生 UI 溢出崩溃。
* **技术重构**：
  * 重构 `home_screen.dart` 的布局。删除原有的固定高度 Container。
  * 引入 `CustomScrollView`，将整个首页的“资产预览”和“校验日志”作为 `SliverList` 放入大滚动区域内，使整个页面可以流畅地上下滑动。
  * 数据为空时，引入统一的 Empty State (空状态) 占位卡片，保证不同数据水位下的高度自适应。
  * 是否考虑将“智能离线扫描识读”和“局域网近场同步”两个按钮优化一下？这尤其是扫描按钮太大了，占空间

### 1.3 资产详情只读视图隔离（解决预览列表点击误入编辑问题）
* **现状**：从首页下方的“本地已下载资产预览”中点击某个工具，会错误地跳转到具备“领用出库”等动作按钮的操作页。在纯查询场景下，用户不需要、也不应该触发修改。
* **技术重构**：
  * 修改 `detail_screen.dart`，在构造函数中引入 `final bool isReadOnly` 参数，默认值为 `false`。
  * 如果 `isReadOnly == true`，利用 `Visibility` 或 `if-else` 条件渲染，彻底屏蔽底部的 【确认领用出库】、【进入归库保养与配件核销】 及 【现场工况井号变更】 三个交互动作按钮。
  * 另外是否需要预留我之前提到的【设备基础信息】空间，当前可以使用mock数据

### 1.4 近场一致性同步控制优化（自动同步与强制提醒机制）
* **现状**：离线状态下操作成功，但有网时需要用户手动去点击“近场同步”才能更新库房端数据，缺乏交互引导与强一致性保障。
* **技术重构（混合同步策略）**：
  * **自动尝试同步**：当用户在扫码或手动操作出库/入库成功时，App 在提交本地库的同时，自动在后台静默发起一次同步请求。
  * **有网状态处理**：如果静默同步成功，直接在成功弹窗提示：`“操作已记录，并已同步更新至库房端。”`。
  * **无网状态处理**：如果静默同步超时或失败，操作会照常安全记入本地 SQLite 数据库，并弹窗强提示：`“当前处于离线状态，操作已记入本地。请在回到局域网环境后，前往首页点击「同步」更新库房中枢！”`。
  * **主页待同步提醒**：在首页同步按钮上增加一个红色的 badge 气泡数字，直观显示当前有几条离线日志还未同步对齐。
