# 项目环境搭建与调试指南

* **日期**：2026-06-23
* **版本**：V1.1
* **编写人**：Antigravity (AI 架构师)

本指南旨在为后续开发人员提供一套标准的环境搭建、启动调试、测试及关停流程，以确保「精密工具智能化管理系统」的多端（App 端、Web 端、后端中枢）联合调试能够快速上手。

## 1. 项目环境搭建

项目由三部分组成：
1. **`proj/server`**：基于 Python FastAPI 的后端同步中枢。
2. **`proj/app`**：基于 Flutter 的手持移动端 App。
3. **`proj/web`**：基于 Vue 3 + TypeScript 的 Web 网页端台账管理后台。

### 1.1 后端服务环境（Python FastAPI）
1. **创建 Conda 虚拟环境**：
   建议使用 Python 3.10 及以上版本。
   ```bash
   conda create -n item_intelli python=3.10
   conda activate item_intelli
   ```
2. **安装核心依赖项**：
   ```bash
   cd proj/server
   python -m pip install -r requirements.txt
   ```
   如果确认本机 `pip` 指向当前 Python 环境，也可以使用 `pip install -r requirements.txt`。
3. **后端环境变量配置**：
   当前后端已经禁止生产环境静默降级 SQLite。启动前必须配置数据库、CORS、JWT 和初始管理员。开发调试可在 PowerShell 中临时设置：
   ```powershell
   cd "E:\ww\sz work\item_intelli\proj\server"
   $env:DATABASE_URL="sqlite:///./item_intelli_dev_test.db"
   $env:ALLOW_SQLITE_FALLBACK="true"
   $env:ALLOWED_ORIGINS="http://localhost:5173,http://127.0.0.1:5173"
   $env:JWT_SECRET_KEY="dev-test-secret"
   $env:INITIAL_ADMIN_USERNAME="admin"
   $env:INITIAL_ADMIN_PASSWORD="admin123"
   ```
   也可以参考 `proj/server/.env.example` 创建本机 `proj/server/.env`。注意 `.env` 不入库。
4. **数据库迁移**：
   后端不再在启动时自动 `create_all()` 建表。首次启动或模型迁移后必须执行：
   ```powershell
   cd "E:\ww\sz work\item_intelli\proj\server"
   python -m alembic upgrade head
   ```
   如果使用真实 PostgreSQL，把 `DATABASE_URL` 改为 PostgreSQL 连接串后再执行迁移。

### 1.2 移动端环境（Flutter）
1. **基础依赖**：
   确保安装好 Flutter SDK (建议与项目兼容的最新 Stable 版本) 以及 Android Studio (或包含 Android SDK 命令行工具的开发环境)。
2. **手机准备**：
   * 使用小米6等 Android 真机，进入「设置 -> 更多设置 -> 开发者选项」。
   * 开启 **「USB 调试」** 和 **「USB 安装（允许通过 USB 安装应用）」**。
   * 用数据线将手机连接至电脑。
3. **验证设备在线**：
   打开命令行，输入以下命令检查设备列表：
   ```bash
   D:\FlutterDev\flutter\bin\flutter.bat devices
   e:\flutter\bin\flutter.bat devices
   ```
   记录对应手机的 `Device ID` (例如：`fa861c00`)。

### 1.3 Web 前端环境（Vue）
1. **环境准备**：
   确保本地安装了 Node.js (建议 18.x 或以上 LTS 版本)。
2. **安装依赖**：
   ```bash
   cd proj/web
   npm install
   ```

## 2. 启动与联合调试步骤

进行多端联调时，请务必遵循以下启动顺序，以保证接口数据通道正常：

### 2.1 启动后端同步中枢
为使处于局域网中的手机 App 能够成功发起连接，启动时**必须监听 `0.0.0.0`**，且端口设置为 `8000`。启动前确认已设置环境变量并执行过 Alembic 迁移。

```powershell
cd "E:\ww\sz work\item_intelli\proj\server"
python -m uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

如果直接执行 `uvicorn main:app ...` 也可以，但要确保它使用的是同一个 Python 环境。若看到“未设置 DATABASE_URL，且 ALLOW_SQLITE_FALLBACK 未启用”，说明当前终端没有配置后端环境变量。

### 2.2 启动 Web 端管理后台
```bash
cd proj/web
npm run dev
```
启动后在电脑端浏览器中打开显示的地址（通常是 `http://localhost:5173`）。

### 2.3 启动 App 移动端并安装至手机
1. 将电脑和手机连接到**同一个 Wi-Fi（局域网）**下。
2. 打开终端，切换至 `proj/app` 目录。
3. 运行调试安装命令（以设备 ID `fa861c00` 为例）：
   ```bash
   D:\FlutterDev\flutter\bin\flutter.bat run -d QDY4C17717003909
   ```
   *注意：此时注意解锁手机，在弹出的“是否允许通过 USB 安装”系统弹窗上点击“允许”。*

## 3. 端到端测试方法（闭环测试用例）

以下是验证「离线采集与近场对齐」核心业务逻辑的标准测试步骤：

### 3.1 步骤一：绑定局域网 API 地址
1. 手机 App 启动成功后，进入 **「系统手持配置」** 页面。
2. 在「中枢服务器 API 地址」输入框中填入**运行后端的电脑局域网 IP 地址**（通过电脑命令行 `ipconfig` 获取，如 `192.168.120.107`），端口为 `8000`：
   * 输入格式：`http://192.168.120.107:8000/api/v1`
   * 不要填写 `127.0.0.1`，安卓真机访问不到电脑本机回环地址。
3. 保存配置。

### 3.2 步骤二：Web 登录与设备授权
1. 访问电脑浏览器上的 Web 后台，默认地址通常为 `http://localhost:5173`。
2. 使用后端环境变量中的初始管理员登录，例如开发调试时为 `admin / admin123`。
3. 进入「终端设备授权」页面。
4. 在 App「系统手持配置」页面查看当前终端 UUID。T4 未完成前，默认可能仍是 `terminal-handheld-001`。
5. 在 Web 端新增该 UUID，并保持启用状态。

### 3.3 步骤三：中枢端（Web）建档
1. 在 Web 工具管理页新建一个精密工具。
2. 填写条码为 `TL-TEST-999`，将其初始状态设为**在库**。

### 3.4 步骤四：App 端拉取并执行离线操作
1. 手机 App 登录进入「近场同步」页，点击「数据拉取 / 一键对齐」。
2. 检查手机端的“本地资产预览”列表中是否成功出现了 `TL-TEST-999` 工具。
3. 手机模拟进入离线状态（可以直接关闭手机 Wi-Fi，或直接使用本地存储）。
4. 在手机 App 首页进入扫码/输入框，输入编码 `TL-TEST-999`，执行 **「领用出库」**，领用人写“李志刚”，目的地写“川科1井”。
   * 观察手机本地操作日志应增加一条待同步记录。

### 3.5 步骤五：近场握手数据对齐
1. 重新打开手机 Wi-Fi 确保与电脑局域网联通。
2. 在手机 App 上进入「近场同步」页，点击 **「开始数据同步」**。
3. 观察同步完成后，App 端与 Web 端控制台是否提示握手对齐成功。
4. 刷新电脑 Web 后台页面，检查工具 `TL-TEST-999` 的状态是否更新为了 **「离库」**，且位置变成了 **「川科1井」**，并且生命周期履历中记录了最新的离线操作详情。

### 3.6 设备授权负向测试
1. 在 Web「终端设备授权」页面禁用当前 App UUID。
2. App 再次点击同步，应收到设备未授权或已禁用相关失败提示。
3. Web 重新启用该设备。
4. App 再次同步，应恢复成功，并且 Web 设备页的最近同步时间应更新。

## 4. 结束与断开步骤

测试完毕后，为防止 Android 调试守护进程挂起或占用系统端口，请严格按照以下步骤退线：

1. **第一步：停止 App 调试（最关键）**
   在运行 `flutter run` 的终端中，按下键盘上的 **`q`** 键（退出调试）。这会触发手机端 Debug 连接的安全释放。
2. **第二步：停止后端与前端服务**
   * 在 `uvicorn` 运行的终端按 **`Ctrl + C`** 结束进程。
   * 在 Web 前端服务终端按 **`Ctrl + C`** 结束。
3. **第三步：断开手机**
   待终端命令行都恢复可输入状态后，**从电脑上拔掉手机 USB 数据线**。
