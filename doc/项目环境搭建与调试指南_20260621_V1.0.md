# 项目环境搭建与调试指南

* **日期**：2026-06-21
* **版本**：V1.0
* **编写人**：Antigravity (AI 架构师)

本指南旨在为后续开发人员提供一套标准的环境搭建、启动调试、测试及关停流程，以确保「精密工具智能化管理系统」的多端（App 端、Web 端、后端中枢）联合调试能够快速上手。

---

## 1. 项目环境搭建

项目由三部分组成：
1. **`proj/server`**：基于 Python FastAPI 的后端同步中枢。
2. **`proj/app`**：基于 Flutter 的手持移动端 App。
3. **`proj/web`**：基于 Vue 3 + TypeScript 的 Web 网页端台账管理后台。

### 1.1 后端服务环境（Python FastAPI）
1. **创建 Conda 虚拟环境**：
   建议使用 Python 3.10 及以上版本。
   ```bash
   conda create -n item_intelli python=3.10
   conda activate item_intelli
   ```
2. **安装核心依赖项**：
   ```bash
   pip install fastapi uvicorn sqlalchemy pydantic spec
   ```
   *(注：Pydantic 需使用 2.0 以上版本)*
3. **数据库依赖**：
   系统默认在首次运行启动时，会在本地 `proj/server/` 目录下自动生成 SQLite 数据库文件 `item_intelli.db`，无需手动配置 SQL 导入。

### 1.2 移动端环境（Flutter）
1. **基础依赖**：
   确保安装好 Flutter SDK (建议与项目兼容的最新 Stable 版本) 以及 Android Studio (或包含 Android SDK 命令行工具的开发环境)。
2. **手机准备**：
   * 使用小米6等 Android 真机，进入「设置 -> 更多设置 -> 开发者选项」。
   * 开启 **「USB 调试」** 和 **「USB 安装（允许通过 USB 安装应用）」**。
   * 用数据线将手机连接至电脑。
3. **验证设备在线**：
   打开命令行，输入以下命令检查设备列表：
   ```bash
   D:\FlutterDev\flutter\bin\flutter.bat devices
   ```
   记录对应手机的 `Device ID` (例如：`fa861c00`)。

### 1.3 Web 前端环境（Vue）
1. **环境准备**：
   确保本地安装了 Node.js (建议 18.x 或以上 LTS 版本)。
2. **安装依赖**：
   ```bash
   cd proj/web
   npm install
   ```

## 2. 启动与联合调试步骤

进行多端联调时，请务必遵循以下启动顺序，以保证接口数据通道正常：

### 2.1 启动后端同步中枢
为使处于局域网中的手机 App 能够成功发起连接，启动时**必须监听 `0.0.0.0`**，且端口设置为 `8000`：
```bash
cd proj/server
uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

### 2.2 启动 Web 端管理后台
```bash
cd proj/web
npm run dev
```
启动后在电脑端浏览器中打开显示的地址（通常是 `http://localhost:5173`）。

### 2.3 启动 App 移动端并安装至手机
1. 将电脑和手机连接到**同一个 Wi-Fi（局域网）**下。
2. 打开终端，切换至 `proj/app` 目录。
3. 运行调试安装命令（以设备 ID `fa861c00` 为例）：
   ```bash
   D:\FlutterDev\flutter\bin\flutter.bat run -d fa861c00 -v
   ```
   *注意：此时注意解锁手机，在弹出的“是否允许通过 USB 安装”系统弹窗上点击“允许”。*

## 3. 端到端测试方法（闭环测试用例）

以下是验证「离线采集与近场对齐」核心业务逻辑的标准测试步骤：

### 3.1 步骤一：绑定局域网 API 地址
1. 手机 App 启动成功后，进入 **「系统手持配置」** 页面。
2. 在「中枢服务器 API 地址」输入框中填入**运行后端的电脑局域网 IP 地址**（通过电脑命令行 `ipconfig` 获取，如 `192.168.120.107`），端口为 `8000`：
   * 输入格式：`http://192.168.120.107:8000`
3. 保存配置。

### 3.2 步骤二：中枢端（Web）建档
1. 访问电脑浏览器上的 Web 后台，在工具管理页新建一个精密工具。
2. 填写条码为 `TL-TEST-999`，将其初始状态设为**在库**。

### 3.3 步骤三：App 端拉取并执行离线操作
1. 手机 App 登录进入「近场同步」页，点击「数据拉取 / 一键对齐」。
2. 检查手机端的“本地资产预览”列表中是否成功出现了 `TL-TEST-999` 工具。
3. 手机模拟进入离线状态（可以直接关闭手机 Wi-Fi，或直接使用本地存储）。
4. 在手机 App 首页进入扫码/输入框，输入编码 `TL-TEST-999`，执行 **「领用出库」**，领用人写“李志刚”，目的地写“川科1井”。
   * 观察手机本地操作日志应增加一条待同步记录。

### 3.4 步骤四：近场握手数据对齐
1. 重新打开手机 Wi-Fi 确保与电脑局域网联通。
2. 在手机 App 上进入「近场同步」页，点击 **「开始数据同步」**。
3. 观察同步完成后，App 端与 Web 端控制台是否提示握手对齐成功。
4. 刷新电脑 Web 后台页面，检查工具 `TL-TEST-999` 的状态是否更新为了 **「离库」**，且位置变成了 **「川科1井」**，并且生命周期履历中记录了最新的离线操作详情。

## 4. 结束与断开步骤

测试完毕后，为防止 Android 调试守护进程挂起或占用系统端口，请严格按照以下步骤退线：

1. **第一步：停止 App 调试（最关键）**
   在运行 `flutter run` 的终端中，按下键盘上的 **`q`** 键（退出调试）。这会触发手机端 Debug 连接的安全释放。
2. **第二步：停止后端与前端服务**
   * 在 `uvicorn` 运行的终端按 **`Ctrl + C`** 结束进程。
   * 在 Web 前端服务终端按 **`Ctrl + C`** 结束。
3. **第三步：断开手机**
   待终端命令行都恢复可输入状态后，**从电脑上拔掉手机 USB 数据线**。
